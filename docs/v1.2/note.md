## app에 있는 언어모델이 툴콜링하는 방식을 langchain과 langgraph, 그리고 tool node를 사용하여 현재 있는 툴을 툴 콜링하여 진행토록 변경합니다. 

## 현재 model/에 data/test.csv를 기반으로 eclo를 예측할 수 있는 모델과 라벨 인코더, feature_config가 들어있습니다. 이를 생성한 주피터 노트북은 material/3-7/3_2_데이콘_데이터_제출해보기.ipynb에 있습니다. 이번 v1.2에선 현재 app에서 채팅으로 각 데이터에 대하여 질의응답할 수 있는 탭의 챗봇에, 자연어로 ECLO를 예측하는 기능을 툴콜링 추가를 통해 적용하려고 합니다. 해당 탭에서 train이나 test 탭이 선택되었을 경우에 한하여, 사용자가 ECLO를 예측하려고 할 때 필요한 정보를 언어모델이 재 질문 하면서 필요한 정보가 모두 모였을 때 ECLO를 예측하는 모델을 돌려서 값을 받아와 사용자에게 자연어로 응답을 내려줘야 합니다. 즉, train이나 test 데이터가 데이터 질의응답 탭에서 활성화 되었을 때 사용자가 eclo를 예측하려고 일부데이터를 주거나 그런 의도를 내비친다면 재질문을 반복해서 모델을 작동시키기 위한 필수 데이터를 다 받고 그게 다 받아지면 모델을 실행시켜 그 답을 바탕으로 자연어로 대답하여 마치 사용자는 채팅으로 모델을 작동시키는 경험을 느끼도록 해야 합니다.
